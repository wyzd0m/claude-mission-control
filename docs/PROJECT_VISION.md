# Project Vision

## Vision statement

Claude Mission Control is a local-first MCP workspace extension that helps people manage complex projects with Claude Desktop while making observable AI-tool activity understandable, useful, and visually engaging.

It combines structured project management with a clean, futuristic, low-poly isometric facility. Claude appears to dispatch small robots into functional rooms when Mission Control tools are used. The facility is a truthful visual metaphor for events the extension can observe, not a representation of Claude's private thoughts.

## Problem

Long-running work with an AI assistant becomes difficult to track:

- Goals drift across conversations.
- Decisions are buried in chat history.
- Tasks are discussed but not saved.
- Users lose track of the current development stage.
- A new conversation lacks a useful handoff.
- Tool activity appears only as raw technical output.
- Local MCP installation is too difficult for nontechnical users.

Mission Control addresses these problems by owning explicit project state and presenting it through both a structured dashboard and an event-driven visual environment.

## Primary user

An individual using Claude Desktop for multi-step work such as:

- Software development
- Game development
- Research
- Content production
- Product planning
- Portfolio projects
- Technical troubleshooting

## Portfolio audience

Recruiters and engineering managers should be able to see evidence of:

- Practical MCP implementation
- Product thinking
- AI workflow design
- Local-first architecture
- Packaging and installation
- UI engineering
- 3D visualization
- Safety and truthful system design

## Core value proposition

> Keep project state organized, create reliable handoffs, and see observable Mission Control activity through a living isometric facility inside Claude Desktop.

## Product principles

### Truth before spectacle

The 3D environment must never imply access to data the extension cannot observe.

### Useful without animation

Every important state also appears as text, structured data, and accessible controls.

### Local first

Project data remains local by default. Version 1 has no cloud service, account system, or telemetry requirement.

### One-click installation

A user should not need to install a runtime or manually edit Claude configuration.

### Modular architecture

The domain, storage, MCP interface, event system, and renderer remain separately testable.

### Calm visual design

The facility should feel active when real work occurs and comfortably idle when it does not.

### Claude-generated execution

The implementation, procedural art, tests, packaging, and documentation are generated through Claude-led development rather than manual 3D asset creation.

## Version 1 promise

Version 1 provides:

- Local projects and project stages
- Tasks, blockers, decisions, artifacts, and checkpoints
- Compact context and handoff summaries
- An embedded interactive dashboard
- An event-driven isometric facility
- Clear operation history and status
- Import and export
- One-click installation
- No API key requirement

## Long-term possibilities

After version 1:

- Full-window local dashboard
- Standalone ambient second-monitor view
- More facility themes
- Workflow templates
- Carefully scoped integrations
- Team or sync features

These are not version 1 commitments.
